# Claude Codeを導入したら「判断する仕事」が増えた話

## はじめに

25人のAI社員チームを運用して数ヶ月。一番大きな変化は、**自分の仕事が「作業」から「判断」に変わったこと**だった。

これは嬉しい変化でもあり、正直しんどい変化でもある。

## 何が変わったか

以前の1日：
- メールの返信を書く
- 資料を作る
- リサーチする
- スケジュールを調整する

今の1日：
- AI秘書ミナミが用意した返信ドラフトを**承認するか修正するか判断する**
- リン（COO）が構造化した選択肢から**方針を決める**
- コトハ（リサーチャー）の調査結果を見て**次のアクションを判断する**

作業時間は激減した。でも**判断の回数は倍以上に増えた**。

## 「判断疲れ」という新しい課題

AI社員が優秀であるほど、判断を求められる頻度が上がる。これは予想していなかった。

1日に50回「Yes/No/修正」を繰り返すと、夕方には頭がぼんやりする。いわゆる「決断疲れ（Decision Fatigue）」だ。

## 対策：判断の階層化

解決策として、**判断の階層化**を導入した。

| レベル | 判断者 | 例 |
|--------|--------|-----|
| L1: 定型判断 | AI社員が自律判断 | メール返信、日報作成、記事リサーチ |
| L2: 方針判断 | リン（COO）が判断 | タスク優先順位、チーム間調整 |
| L3: 戦略判断 | 千葉（代表）が判断 | 新規事業、外部パートナーシップ、ブランド方針 |

これにより、自分がやるべき判断は **L3の戦略判断だけ** になった。

## まとめ

AI社員を導入して得たのは「時間」ではなく「判断に集中できる環境」だった。

作業を手放すのは簡単だ。でも、判断の質を上げるには、自分自身が成長し続ける必要がある。

AIは魔法じゃない。加速装置だ。加速した分、自分の判断力も磨かなければいけない。
