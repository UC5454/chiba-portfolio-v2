# メディア記事 投稿ガイド

千葉勇志ポートフォリオサイトのメディア記事を追加・編集するための手順書。
AI社員（ツムギ、コトハ等）が記事を投稿する際にも参照すること。

---

## 記事の追加手順

### 1. MDファイルを作成

配置場所: `src/content/media/{カテゴリslug}/{記事slug}.md`

```
src/content/media/
  ai-employees/     # AI社員・実運用
  ai-business/      # AI導入・ビジネス
  education/        # AI研修・教育
  government/       # 行政・自治体連携
  column/           # コラム
  events/           # イベント・コミュニティ
  tech/             # テクニカル
```

カテゴリディレクトリが存在しない場合は作成する。

### 2. ファイル命名規則

- **英語のケバブケース**（小文字、ハイフン区切り）
- ファイル名がそのままURLスラッグになる
- 日付は入れない（front matterのdateで管理）
- 内容がわかる簡潔な名前にする

```
OK: claude-code-changed-my-decisions.md
OK: 25-ai-employees-result.md
NG: 2026-03-01-記事.md
NG: article_01.md
```

### 3. テンプレートをコピー

`src/content/media/_template.md` をコピーして編集する。

## Front Matter リファレンス

```yaml
---
title: "25人のAI社員を雇った結果、起きたこと"
description: "25名6チームのAI社員組織を構築・運用して見えた、リアルな変化と学び。"
date: "2026-03-01"
updatedAt: "2026-03-10"
thumbnail: "/images/media/25-ai-employees.webp"
category: "ai-employees"
tags: ["AI社員", "AIエージェント", "Claude Code", "業務効率化"]
---
```

| フィールド | 必須 | 型 | 説明 |
|-----------|------|-----|------|
| `title` | Yes | string | 記事タイトル。30-60文字推奨 |
| `description` | Yes | string | 記事の要約。80-120文字推奨。検索結果のスニペットに表示される |
| `date` | Yes | string | 公開日。`YYYY-MM-DD` 形式 |
| `updatedAt` | No | string | 更新日。更新した場合のみ記入 |
| `thumbnail` | No | string | サムネイル画像パス。1200x630px、WebP形式推奨 |
| `category` | Yes | string | 下記カテゴリ一覧のslugを指定 |
| `tags` | Yes | string[] | 3-5個推奨。SEOキーワードを含める |

## 記事の書き方

### 見出し構造

- `##`（H2）で大見出し、`###`（H3）で小見出しを使う
- `#`（H1）は使わない。titleがH1として自動表示される
- 見出しには自動でIDが付与され、目次リンクに使われる

### 画像

- 配置場所: `public/images/media/`
- 形式: **WebP推奨**
- 記事内での参照: `![代替テキスト](/images/media/filename.webp)`
- サムネイル推奨サイズ: 1200x630px（OGP対応）

### 読了時間

本文の文字数から自動計算される（約500文字/分）。手動設定は不要。

## AI社員向けの注意点

### ツムギ（ライター）・コトハ（リサーチャー）へ

- 記事のトーン&マナーは千葉さんのブランドガイドラインに準拠すること
- 自信と実証に基づく語り口。対比構造で本質を突く。泥臭い失敗も率直に語る
- 「体験主義」「加速装置」「Winning Strategy」のキーワードを自然に織り込む
- 事実に基づかない情報は書かない。数値・事例には出典を明記する
- 記事末尾に `*千葉勇志 — 株式会社SOU 代表 / デジタルゴリラ 取締役COO兼CAIO*` を入れる

### 投稿フロー

1. MDファイルを作成し、front matterを正確に記入
2. カテゴリslugは上記一覧から選ぶ（新規カテゴリの追加はカイトに相談）
3. サムネイル画像が必要な場合はソラ（デザイナー）またはアヤ（画像生成）に依頼
4. 完成したらカイトのINBOXに報告し、デプロイを依頼

## SEOチェックリスト

記事公開前に以下を確認すること。

- [ ] **title**: 30-60文字。主要キーワードを前半に配置しているか
- [ ] **description**: 80-120文字。記事内容の要約になっているか。検索結果で魅力的に見えるか
- [ ] **見出し構造**: H2→H3の階層が正しいか。H1を使っていないか
- [ ] **タグ**: 3-5個設定しているか。検索されそうなキーワードを含んでいるか
- [ ] **内部リンク**: 関連する他の記事へのリンクを入れているか
- [ ] **画像alt属性**: 全ての画像に代替テキストを設定しているか
- [ ] **ファイル名**: ケバブケースの英語で、URLとして自然か
- [ ] **カテゴリ**: 適切なカテゴリを選択しているか（front matterとディレクトリが一致）
